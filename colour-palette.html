Stroke & Outline Guide

Stroke widths are in SVG user units (px at 1:1 scale) for SVG tools, and canvas lineWidth logical pixels for PV. PV renders at scale ×2 (preview) and ×3 (QTI export), so its lineWidths appear proportionally heavier in the final output.

Inconsistencies found in current code
Previously resolved inconsistencies. BM segment dividers were 2.5 (heavier than the outer outline); fixed to 2.0. NR bead string was 2.2 (non-standard); fixed to 2.0. NR bead circle outlines were 1.0 (lighter than all other NR shapes); fixed to 2.0. All changes are in tools/shared.js.
Number Track — relative stroke widths. NT stroke widths scale with cell size (CELL×0.022 / CELL×0.029) rather than using fixed values like all other SVG tools. This is intentional — shapes look proportional at any size — but the approach differs from NL, BM, and NR which use fixed px. No action needed; worth knowing when adding new tools.
Place Value — canvas uses #111 for structural outlines. PV uses near-black (#111) for block grid and outlines rather than colour-matched or palette strokes. This is consistent with physical manipulative aesthetics (pencil-line grid on coloured blocks) and is intentional. PV is also the only canvas-rendered tool, so direct weight comparison with SVG tools is not straightforward.
